🔄 Ring Buffer (Circular Buffer)

Fixed-capacity, mutable circular buffer with O(1) push/pop from both ends

Overview

Implements a ring buffer (circular buffer) — a fixed-size data structure that wraps around when full, overwriting the oldest elements. Ideal for streaming data, bounded queues, and sliding window algorithms.

Concepts Demonstrated

  • Mutable state — uses OCaml's mutable record fields and arrays
  • Modular arithmetic — index wrapping via mod
  • Module system — encapsulated API via module RingBuffer
  • Higher-order functions — iter, fold, map, exists, for_all
  • Polymorphic containers — generic 'a t type
  • Overflow semantics — oldest elements silently replaced when full

Try It

(* Create a buffer of capacity 5 *)
let rb = RingBuffer.create 5 0

(* Fill it up *)
let () = List.iter (RingBuffer.push_back rb) [10; 20; 30; 40; 50]
(* rb = [|10; 20; 30; 40; 50|] *)

(* Push more — oldest elements are overwritten *)
let () = RingBuffer.push_back rb 60
(* rb = [|20; 30; 40; 50; 60|]  — 10 was dropped *)

(* Pop from front *)
let v = RingBuffer.pop_front rb  (* v = 20 *)

(* Fold to sum *)
let sum = RingBuffer.fold_left ( + ) 0 rb  (* 30+40+50+60 = 180 *)

Use Cases

  • Log buffers — keep last N log entries in memory
  • Audio/video streaming — bounded producer-consumer buffer
  • Sliding window algorithms — moving average, rate limiting
  • Undo history — fixed-depth undo with automatic oldest-discard
  • Network I/O — packet buffering with bounded memory
